# 设备接入案例（多产品形态）

## 案例 A：ESP32 + HUB75（创客与中小批量）

适用：

- 个人开发者
- 教育与创客产品

特点：

- 资料最成熟
- 成本低
- 开发快

建议：

- 先实现 `0x01/0x02/0x03`
- 再加心跳和鉴权

## 案例 B：nRF52 + 自研驱动板（低功耗产品）

适用：

- 电池供电产品
- 强调 BLE 连接质量的终端

特点：

- BLE 稳定
- 功耗更低
- 驱动开发成本更高

建议：

- GATT 层先完整通过
- 再接显示驱动

## 案例 C：Linux 网关 + 外设板（系统集成）

适用：

- 工业或展陈场景
- 多设备集中管理

特点：

- 便于远程运维
- 容易集成日志与监控
- 需要处理网关稳定性

建议：

- 使用统一重试机制
- 给每台设备维护连接状态机

## 统一接入基线

- 必须声明协议版本
- 必须限制写入速率
- 必须处理断连恢复
- 商用前必须通过兼容性认证
