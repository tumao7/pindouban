# 04-安全与稳定要求

## 1. 安全基线（必须）

- 禁止未鉴权直接执行高影响命令（建议 V1.1 起强制）
- 固件必须限制写入频率
- 异常包必须丢弃，禁止越界写显存
- 断连后必须重置会话状态

## 2. 推荐安全机制

### 2.1 会话鉴权

- 小程序发 `nonce`
- 设备返回签名
- 双方建立短时会话密钥

### 2.2 设备身份

- 固件内置设备型号与协议版本
- 小程序校验设备声明字段后再开放控制

### 2.3 命令限流

- 每秒最大包数限制
- 连续超限触发软封禁（例如 3 秒）

## 3. 稳定性基线（必须）

- 连续运行 30 分钟无崩溃
- 重复连接 100 次无死锁
- 异常断连后 3 次内可恢复

## 4. 兼容性建议

- Android / iOS 都需实测
- MTU 协商失败时自动降包长
- 不同分辨率设备要返回有效尺寸信息

## 5. 性能建议

- 绘制前做坐标去重
- 合并同色点位，减少包数量
- 使用批量刷新替代逐点刷新

## 6. 失败处理

统一错误策略：

- 可恢复错误：自动重试
- 不可恢复错误：提示用户重连
- 安全错误：直接断开并记录日志
