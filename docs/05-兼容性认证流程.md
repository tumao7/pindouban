# 05-兼容性认证流程

## 1. 认证目标

保证第三方设备在小程序中的可用性、稳定性和安全性，降低用户接入风险。

## 2. 认证分级

- `Compatible-L1`：通过基础功能测试（清屏/绘制/显示）
- `Compatible-L2`：通过稳定性与重连测试
- `Compatible-L3`：通过安全与高压测试（推荐商用）

## 3. 提交流程

1. 提交设备信息清单（型号、MCU、分辨率、固件版本）
2. 提交协议自测报告
3. 提交实测视频（连接、点亮、断连恢复）
4. 平台执行回归测试
5. 出具兼容等级结果

## 4. 必交材料

- 设备说明文档
- 接线图或原理图摘要
- BLE 广播与服务声明
- 错误码实现说明
- 固件版本号命名规则

## 5. 回归测试项

- 基础命令正确性
- 高频点亮稳定性
- 多机型连接一致性
- 会话安全控制
- 异常恢复能力

## 6. 不通过常见原因

- 没有版本字段
- 越界坐标未防护
- 断连后无法恢复
- 高频写入出现卡死
- 无鉴权且无速率限制

## 7. 认证后要求

- 版本升级必须更新变更记录
- 破坏兼容变更需重新认证
- 重大故障需 72 小时内给出修复计划
