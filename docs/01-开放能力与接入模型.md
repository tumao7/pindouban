# 01-开放能力与接入模型

## 1. 目标

本开放能力用于让第三方硬件设备与小程序“拼豆模式”联动，核心场景：

- 小程序选色
- 硬件板对应点亮
- 清屏、局部更新、整帧提交
- 可扩展动画/状态显示

## 2. 角色定义

- 平台方：维护小程序、协议版本、认证标准
- 设备方：实现 BLE 服务与协议解析
- 用户：在小程序内连接设备并进行点亮互动

## 3. 接入模式

### 模式 A：协议直连（推荐）

- 第三方完全自研硬件与固件
- 只按协议实现服务/特征/命令
- 最灵活，适合量产厂商

### 模式 B：参考固件二次开发

- 基于官方示例固件改造
- 成本低，上线快
- 适合个人开发者与中小团队

## 4. 能力分级

- L1 基础能力：连接、清屏、绘制、显示
- L2 增强能力：分包 ACK、心跳、重连
- L3 安全能力：握手鉴权、会话令牌、异常拦截

建议认证至少达到 L2，商用建议达到 L3。

## 5. 不限制项

以下实现自由，不做硬性限制：

- 芯片型号
- 显示驱动方案
- 屏幕类型与分辨率
- 硬件电源与结构设计

## 6. 强约束项

以下必须满足：

- 使用平台要求的 BLE Service/Characteristic 格式
- 支持协议版本字段
- 支持错误码返回或等效状态回传
- 通过稳定性测试清单
