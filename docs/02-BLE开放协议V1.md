# 02-BLE开放协议V1

## 1. 协议版本

- 当前版本：`1.0.0`
- 包头建议：`[ver, cmd, ...payload]`
- `ver` 取值建议：`0x01`

## 2. BLE 基础定义

- Service UUID：`4fafc201-1fb5-459e-8fcc-c5c9c331914b`
- Characteristic UUID：`beb5483e-36e1-4688-b7f5-ea07361b26a8`
- Characteristic 属性：`WRITE` + `WRITE_NO_RESPONSE` + `NOTIFY` (或 `INDICATE`)

设备名格式：
`kmpd-XXXX`
`pindou-XXXX`
命名格式必须任选其一，否则将无法搜索的到

## 3. 指令集 V1

### 3.1 清屏

```text
[0x01]
```

### 3.2 绘制像素

```text
[0x02, R, G, B, X1, Y1, X2, Y2, ...]
```

- `R/G/B`: 0~255
- `Xi/Yi`: 像素坐标
- 单包可携带多个点位

### 3.3 提交显示

```text
[0x03]
```

### 3.4 亮度调节

```text
[0x04, Value]
```

- `Value`: 0~100 (亮度百分比)
- 设备端应在收到此指令后实时或渐变应用亮度，无需等待 `0x03` 显示指令。

## 4. 推荐扩展指令（V1.1+）

### 握手请求

```text
[0x10, nonce(8 bytes)]
```

### 握手响应

```text
[0x11, sign(16/32 bytes)]
```

### 心跳

```text
[0x13, timestamp(4 bytes)]
```

### 4.1 设备按键事件 (上行指令)

```text
[0x05] // 触发事件：切换上一个颜色
[0x06] // 触发事件：切换下一个颜色
```

- 硬件端需对实体按键做防抖处理（建议20~50ms）。
- 硬件端通过 `NOTIFY` 特征值主动向小程序发送此指令。

## 5. 错误码约定

- `0xE1`: 未授权
- `0xE2`: 包格式错误
- `0xE3`: 越界坐标
- `0xE4`: 频率超限
- `0xE5`: 版本不兼容

## 6. 分包建议

- 单包建议不超过 180 字节
- 大图分片发送，分片顺序必须可重组
- 每批次发送结束后调用 `0x03`

## 7. 向后兼容策略

- V1 命令长期保留：`0x01/0x02/0x03`
- 新能力通过新 cmd 添加，不覆盖旧 cmd 语义
- 破坏兼容变更必须升级大版本
