# 03-小程序对接指南

## 1. 接入前提

- 设备支持 BLE 4.2+
- 设备已实现 [02-BLE开放协议V1](file:///d:/xiaochengxu/axcx/hardware/open-smart-pixel-board/docs/02-BLE开放协议V1.md)
- 设备广播名符合规则（建议 `pindou-XXXX`,`kmpd-XXXX`）

## 2. 标准流程

1. 打开蓝牙适配器
2. 扫描设备（按名前缀过滤）
3. 建立 BLE 连接
4. 获取 Service 与 Characteristic
5. 协商 MTU（失败则降级）
6. 发送清屏命令
7. 发送绘制数据（可分包）
8. 发送提交显示命令
9. 维持心跳与连接状态

## 3. 推荐时序

```text
connect -> discover -> mtu -> handshake(optional) -> clear -> draw* -> show
```

## 4. 重连策略

- 异常断连后，最多自动重连 3 次
- 每次重连间隔：500ms / 1000ms / 1500ms
- 重连成功后先清屏再恢复状态

## 5. 发送节流建议

- `WRITE_NO_RESPONSE` 模式下建议 10~30ms 节流
- 批量点位建议按“行块”或“区域块”分发
- UI 操作高频时采用 debounce

## 6. UI 体验建议

- 明确显示连接状态：未连接/连接中/已连接/异常
- 发送中给出 loading 状态
- 失败给出错误码与可执行操作（重试/重连）

## 7. 多硬件厂商适配建议

- 在小程序侧维护“设备能力表”
- 按协议版本与能力位动态启用功能
- 对不支持扩展能力的设备自动降级

示例能力位：

- `bit0`: clear
- `bit1`: draw
- `bit2`: show
- `bit3`: heartbeat
- `bit4`: auth
